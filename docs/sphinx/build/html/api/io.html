I/O¶

The interfaces module contains functions for interfacing with other codes and files.

fire.interfaces.interfaces.generate_camera_id_strings(id_strings, lens, t_int)¶

Return standardised ID strings used for consistency in filenames and data labels

Parameters:
  • id_strings – Dict of string_ids to update/populate
  • lens – Lens on camera
  • t_int – Integration time of camera in seconds
Returns:

Dict of ID strings
fire.interfaces.interfaces.generate_frame_id_strings(id_strings, frame_no, frame_time)¶

Return ID strings for specific analysis frame

Parameters:
  • id_strings – Dict of string_ids to update/populate
  • frame_no – Frame number from movie meta data
  • frame_time – Frame time from movie meta data
Returns:

Dict of ID strings
fire.interfaces.interfaces.generate_pulse_id_strings(id_strings, pulse, camera, machine, pass_no=0)¶

Return standardised ID strings used for consistency in filenames and data labels :param id_strings: Dict of string_ids to update/populate :param pulse: Shot/pulse number or string name for synthetic movie data :param camera: Name of camera to analyse (unique name of camera or diagnostic code) :param machine: Tokamak that the data originates from :return: Dict of ID strings

fire.interfaces.interfaces.identify_input_files(pulse, camera, machine)¶

Return dict of paths to files needed for IR analysis

Parameters:
  • pulse – Shot/pulse number or string name for synthetic movie data
  • camera – Name of camera to analyse (unique name of camera or diagnostic code)
  • machine – Tokamak that the data originates from
Returns:

Dict of filepaths
